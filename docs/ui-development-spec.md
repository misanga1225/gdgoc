# Aurlum 初回UI開発仕様書 v1
## 対象: D-05 説明支援メイン画面

## Summary
本仕様書は、Aurlum の初回UI開発で実装対象とする D-05 説明支援メイン画面について、Vanilla TypeScript で実装着手できる粒度まで具体化した開発用仕様書である。

初回開発では、D-05 をモック専用の独立画面として扱い、README に記載の将来的なライブ監視・AI要約・最終同意フローとは切り分ける。

## 1. 目的
- 医師が資料を主役として閲覧しながら、患者の閲覧結果を補助情報として確認できる画面を作る
- UIは中立的に判断材料を提示し、医療判断や行動提案は行わない
- 初回開発では、見た目・状態・基本操作の成立を優先する

## 6. 表示要素
### 6-1. TopHeader
#### 目的
- 画面識別
- 対象患者識別
- 閲覧状態の表示

#### 表示項目
- `Aurlum` 固定表示
- 患者名
- 患者カルテID
- 閲覧状態

#### 表示しない項目
- セッションID

#### 閲覧状態文言
- `閲覧中`
- `不在`

#### 初回方針
- 閲覧状態は mock の文字列をそのまま表示する
- `不在` は表示確認のみ行い、専用UI分岐は作らない

### 6-2. HtmlDocumentViewer
#### 目的
- 左カラム全体の表示
- 資料タイトルと本文の表示
- 非注視マーカー重ね表示

#### 表示内容
- ビューア上部の資料タイトル
- HTML本文
- 見出し、段落、箇条書き、図のプレースホルダー
- 非注視マーカー

#### 表示ルール
- 資料タイトルはビューア上部に本文と別表示する
- 本文はスクロール型Webドキュメントとして見せる
- 本文最大幅は 760〜860px 程度
- 非注視マーカーは `showAttentionMarkers = true` のときだけ表示する

### 6-3. ViewingStatusPanel
#### 目的
- 更新前空状態と更新後結果表示

#### 更新前
- 文言のみ表示
- `更新ボタンを押すと閲覧状況を確認できます`

#### 更新後
- 経過時間
- 進捗バー + 数値
- 非注視一覧

#### 非注視一覧ルール
- 最大3件表示
- タイトルのみ表示
- 中立的な内容名にする

### 6-4. BottomActionBar
#### 構成
- 左: セッション終了
- 右: 閲覧状況を更新

#### ルール
- 初期状態でも終了ボタンは押せる
- モーダル表示中は背景操作不可
- 初回実装では、更新押下後は即時に更新後状態へ遷移する

### 6-5. Modal
#### 用途
- 終了確認

#### ルール
- キャンセルで閉じる
- 初回は Esc で閉じない
- 初回は外側クリックで閉じない
- 確認押下時も、初回は閉じるだけ
- 背景操作不可

## 8. 状態定義
### UI状態
- `idle`
  - 初期状態
  - 右パネル空状態
  - マーカー非表示
- `updated`
  - 更新後状態
  - 右パネル結果表示
  - マーカー表示
- `modal_open`
  - 終了確認モーダル表示中
- `error`
  - 初回は専用遷移しない
  - 必要なら補助メッセージ追加

### 閲覧状態
- `閲覧中`
- `不在`

## 9. 状態遷移
- 初期表示: `idle`
- 更新ボタン押下: `idle -> updated`
- 終了ボタン押下: `idle -> modal_open` または `updated -> modal_open`
- モーダルキャンセル: `modal_open -> 直前状態へ戻る`
- モーダル確認: `modal_open -> 直前状態へ戻る`
- `不在` は UI状態ではなくヘッダー表示値として扱う

## 18. Phase2 追加仕様（D-02 セッション一覧・ハブ画面）
本章は Phase2 で追加する `D-02` の仕様を定義する。`D-02` はセッション一覧画面であり、`D-03` と `D-05` へのハブ機能を持つ。

### 18-1. 目的
- 医師が患者セッションを一覧管理し、対象患者のファイル確認と次画面遷移を行えるようにする。

### 18-2. レイアウト
- 3ペイン構成（左:中央:右 = `30:50:20`）。
- 左ペイン: 患者一覧。
- 中央ペイン: 選択患者のファイル一覧テーブル（本文プレビューは表示しない）。
- 右ペイン: 資料追加導線（D-03遷移）。

### 18-3. 左ペイン（患者一覧）
- 表示項目: `氏名（+カルテID）`、`状態（閲覧中/不在）`。
- 検索: 患者名+カルテIDの部分一致。
- 初期表示: 一覧が1件以上ある場合は先頭患者行を自動選択する（`selectedSessionId = 先頭行ID`）。
- 初期ファイル選択は未選択（`selectedFileId = null`）とする。
- 検索確定後（Enter）: 自動選択を行わず未選択を維持する。
- 状態変換: `watching` は `閲覧中`、それ以外は `不在`。
- 操作:
  - 単クリック: 患者選択のみ。
  - ダブルクリック: 単クリックと同じ挙動（患者選択のみ）。

### 18-4. 中央ペイン（ファイル一覧）
- ヘッダー: `{選択患者名}のファイル` と `選択患者のファイル総数`。
- 列: `ファイル名`、`更新日時`、`種類`、`サイズ`。
- 操作:
  - 単クリック: ファイル選択。
  - ダブルクリック: D-05へ遷移（正式導線）。
  - Enterキー: D-05へ遷移（キーボード導線）。
- 操作欄アイコン:
  - 操作欄の左アイコン（元ファイル閲覧）は全行で常時表示する。
  - 左アイコンをダブルクリックしたとき、原本ファイルをブラウザ新規タブで表示する。
  - 操作欄の右アイコン（削除）は既存どおり削除操作に使う。
- 削除:
  - 単クリック選択時にゴミ箱アイコンを表示。
  - ゴミ箱押下時は確認モーダルを表示。
  - 削除成功後は一覧即時更新+件数再計算。

### 18-5. 右ペイン（資料追加）
- 右ペイン見出しは `Wordファイルアップロード` とする。
- `資料を追加` ボタンは常時有効。
- 遷移ルール:
  - 患者/ファイル選択あり: D-03へ遷移し、`患者名 + カルテID + 選択セッションID + 選択ファイルID` を渡す。
  - 未選択: 新規患者セッションを作成してD-03へ遷移する。

### 18-7. 一覧データ取得方針
- D-02 の患者セッション詳細取得は逐次ではなく並列取得を基本とする。
- 一部取得失敗時は失敗行のみフォールバック表示し、一覧全体の表示を継続する。

### 18-6. D-05導線の運用
- D-02の左サイドバーにあった `D-05` ボタンは削除する。
- D-05遷移の正式導線は中央ペインのファイル行（ダブルクリック / Enter）とする。
- 左ペインの患者行ダブルクリックではD-05へ遷移しない。
- D-05表示中は左サイドバーを表示しない。
- D-05には `D-02へ戻る` ボタンを置く（TopHeaderのAppBrand隣）。
- 最新URL解決を使う遷移はバックエンド統合後に実装する。Phase2では遷移枠のみ実装対象とする。
- D-05遷移時は `selectedFileId` と `selectedFileName` を遷移ペイロードに含める（今回は受け渡しのみで、D-05表示利用は行わない）。

## 19. Phase2 追加仕様（D-03 資料追加・URL発行）
本章は D-03 のレイアウト・入力・処理・遷移に関する確定事項を定義する。

### 19-1. レイアウト方針
- D-03は単一メイン画面とする（左サイドバーは表示しない）。
- 画面上部に `一覧画面へ戻る` を配置し、メイン領域に資料追加フォームを表示する。
- フォーム本体は固定最大幅（例: `840px`）で扱う。

### 19-2. D-02との関係
- D-02 から D-03 に遷移した際、D-03表示中は左サイドバーを表示しない。
- D-03 のアップロード完了後は D-02 へ戻る。

### 19-3. セッション引き継ぎとアップロード先
- D-03遷移時に `selectedSessionId / selectedFileId` を引き継ぐ。
- `selectedSessionId` が指定されている場合、アップロード先は当該セッションとし、新規セッションは作成しない。
- `selectedSessionId` が未指定の場合のみ、新規セッションを作成してアップロードする。

### 19-4. D-03完了後にD-02へ戻る際の選択復元
- `selectedSessionId` 指定ありでD-03へ遷移した場合: D-02復帰時は当該患者行を再選択する。
- その場合もファイル選択は復元せず、`selectedFileId = null` とする。
- `selectedSessionId` 未指定でD-03へ遷移した場合: D-02復帰時は新規作成されたセッション行を選択する。
- その場合もファイル選択は `selectedFileId = null` とする。

### 19-5. 入力制約
- 受け付ける資料ファイルは `.docx` のみとする。
- ファイルサイズ上限は `50 * 1024 * 1024 byte`（50MiB）とする。
- サイズ判定は `<= 50 * 1024 * 1024` を許可し、超過時のみエラーとする。
- 拡張子判定は大文字小文字を区別せずに行う（`.docx` / `.DOCX` などを許可）。
- 上記制約を満たさないファイルはアップロード処理を開始しない。

### 19-6. バリデーション表示と操作
- 制約違反（拡張子不正・サイズ超過）は、D-03の入力直下にエラーを表示する。
- エラー表示中は送信ボタンを無効化する。
- 正しい `.docx` を再選択した時点でエラー表示を解除し、プレビューを更新する。
- エラー文言の文面は固定しないが、原因が利用者に判別できる意味一致を必須とする。

### 19-7. selectedFileId の扱い
- `selectedFileId` が引き継がれていても、D-03でのアップロードは差し替えではなく常に追加として扱う。
- Phase2 では `selectedFileId` を差し替え更新ロジックには使用しない。

### 19-8. 成功時の表示と遷移
- 成功時の通知文言は `アップロードが完了しました` に統一する。
- 新規セッション作成時のみ患者URLダイアログを表示する。
- 既存セッションへの追加時は患者URLダイアログを表示せず、D-02へ戻る。
