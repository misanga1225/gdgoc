# アプリ名：Aurlum (オーラム)

## 1. 概要
医師がアップロードした同意書・契約書を患者が閲覧する際、**MediaPipeによる視線検知**と**Rustによるハッシュチェーン**を組み合わせ、医療同意の「質の高さ」と「改ざん不能な証跡」を担保するシステム

- 「読んだふり」を防ぎ、説明責任の質を上げ、それを証跡として残す
- オンライン診療下での「リアルタイム説明補助」に対応。医師は患者の注視状況をリアルタイムで把握し、適切なタイミングで補足説明可能
- 個人情報（PII）は削除可能にしながら、証跡（ハッシュ）のみを永続保存する「二層式DB」を採用。Vertex AI経由のGeminiを利用し、入力がモデル学習に利用されない構成

## 2. 技術スタック
- **医師側アプリ:** Tauri (Rust + Vanilla TypeScript)
- **患者側ブラウザ:** Vanilla TypeScript, HTML, CSS
- **視線検知:** MediaPipe
- **リアルタイム同期:** Firestore (`onSnapshot` による双方向同期)
- **文書処理:** Mammoth.js (DOCXからID付HTMLへの変換)
- **バックエンド:** Cloud Run (container, Rust)
- **データベース:** Firestore（実データと永続データで分ける）
- **ストレージ:** Cloud Storage (HTMLドキュメントの管理)
- **セキュリティ:** Cloud KMS (医師の鍵によるデジタル署名)
- **AI:** Vertex AI経由の Gemini 1.5 Flash

## 3. システムフロー

### STEP 1: 【医師】文書準備とセッション発行
1. 医師がTauriアプリにDOCXをアップロード。内部で段落ID付HTMLへ変換
2. 医師が「患者名」「カルテID」「有効期限」を入力
3. **DB保存:** Firestoreの `Patients` に初期記録
4. **URL発行:** UUIDを含む署名型URLを生成し、患者へ送信

### STEP 2: 【患者】閲覧とリアルタイム同期
1. 患者がURLを開き、キャリブレーションを経て閲覧開始
2. **リアルタイム追跡:** MediaPipeが視線を追跡。以下のデータを2〜3秒間隔でFirestoreへ同期
    - **累積注視時間 (`dwell_time`):** 段落ごとの滞在秒数
    - **到達フラグ (`is_reached`):** スクロールによってその段落が画面内に入ったか
3. 患者が読み終えたら、**「仮確認完了（医師へ見落とし箇所を送る）」**ボタンをクリック

### STEP 3: 【医師】リアルタイム監視と最終同意の「ロック解除」
1. **ライブ** 医師は患者の閲覧状況をリアルタイム監視
2. **見落とし判定:** - `is_reached == true`（通り過ぎた） かつ `dwell_time < 閾値`（注視不足）の段落を「読み飛ばし（赤枠）」として強調
    - **後半の未到達部分は無視**し、ノイズを減らす
3. **説明:** 医師が赤枠箇所を口頭説明し、患者が再度そこを注視すると、**赤枠がリアルタイムで緑に変化**
4. 全ての重要項目が「既読」になったことを確認し、医師が**「最終同意を許可」**をクリック

### STEP 4: 【患者】最終同意と証拠の固定（オンチェーン）
1. 患者の画面にリアルタイムで**「最終同意」**ボタンが有効化される
2. 患者が「最終同意」をクリックすると、最終ハッシュが Cloud Run へ送信される
3. **署名・記録:** Cloud Run がハッシュを検証し、**Cloud KMS** で署名。その後、Firestoreの `Evidence` に保存
4. **完了:** 患者画面に「同意照会番号」を表示し、URLを即時無効化
5. **（将来的な展望）:** 蓄積された`Evidence` のルートハッシュを、定期的にブロックチェーン（Polygon等）にアンカーとして記録（展望）

## 4. 医師側ダッシュボード (ライブ・ビュー)
オンライン診療中でも直感的に状況把握できるUI。

### ライブ・インジケーター
- 医師が「表示する」ボタンを押した時だけ，ドキュメント上に患者の見落とし個所が赤枠で囲われて表示される
- vertex経由のgeminiが見落とし箇所をまとめて医師に伝える

| 患者名 | ステータス | 進捗 | アクション |
| :--- | :--- | :--- | :--- |
| **佐藤 太郎** | **[ライブ閲覧中]** | 65% (見落とし有) | **[ライブ画面を開く]** |
| **鈴木 花子** | **[完了]** | 100% | **[証跡を表示]** |

## 5. データベース設計 (Firestore)

### `Patients` コレクション (削除可能)
患者ごとの動的な状態管理
- `session_id`: string (PK)
- `name`: string
- `patient_id`: string
- `status`: string (waiting / watching / reviewed / authorized / completed)
- `current_scroll_pos`: float (0.0 ~ 1.0)
- **`LiveGaze` サブコレクション (リアルタイム同期用)**
    - `paragraph_id`: string (DOCX変換時のID)
    - `dwell_time`: float (累積注視時間/秒)
    - `is_reached`: boolean (到達済みか)
    - `last_updated`: timestamp

### `Evidence` コレクション (永続保存)
削除権の影響を受けない、証拠データのみを保持
- `session_id`: string (PK)
- `root_hash`: string (最終確定ハッシュ)
- `kms_signature`: string
- `blockchain_tx_hash`: string
- `timestamp`: timestamp
