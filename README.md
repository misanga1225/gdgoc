## 1. 概要
医師が患者に送信した同意書・契約書を患者が閲覧しながら診療をする**オンライン診療**において、医療合意の「質の高さ」と「改ざん不能な証跡」を担保することを目指すシステム。
同意書・契約書における患者の見落とし個所を数理モデルで検知して医師に通知する。
患者側は、重要なのに見落としていた箇所の説明を医師から受けることができ、自身の症状や治療、薬について理解した上で同意することができる。
医師側も自身が説明責任を果たした証跡を残すことができ、説明責任を果たした/果たしてないという法的な争点を解消できる。
医師側は不正に患者の見落とし個所を偽造できない。また、医師、患者どちらも同意後のデータを改ざんすることはできない。

## 2. 技術スタック
- **医師側アプリ:** Tauri (Rust + Vanilla TypeScript)
- **患者側ブラウザ:** Vanilla TypeScript, HTML, CSS
- **視線検知:** MediaPipe
- **リアルタイム同期:** Firestore (双方向同期通信)
- **バックエンド:** Cloud Run (Rust)
- **データベース:** Firestore（実データとオンチェーンにする永続データを分けることで、削除権を担保しつつ証跡を残す構造）
- **ストレージ:** Cloud Storage (HTMLドキュメントの管理)
- **セキュリティ:** Cloud KMS (医師の鍵によるデジタル署名)
- **AI:** Vertex AI経由で学習に使われないようにした Gemini 2.5 Flash

## 3. フロー

### STEP 1: 【医師】文書準備とセッション発行
1. 医師がTauriアプリで「患者名」「カルテID」「患者のメールアドレス」を入力してドキュメントをアップロード。**内部で段落ID付HTMLへ変換**
2. DB保存: Firestoreの実データ層に初期記録
3. URL発行: 署名型URLを生成し、既存の病院側が採用しているメールサービスを用いて患者へ送信

### STEP 2: 【患者】閲覧とリアルタイム同期
1. 患者がURLを開き、メールアドレスで**患者本人であることを認証**した後、キャリブレーションを経て閲覧開始
2. リアルタイム追跡: MediaPipeが視線を追跡。以下のデータを2〜3秒間隔でFirestoreへ同期
    - 累積注視時間: 段落ごとの滞在秒数
    - 到達フラグ: スクロールによってその段落が画面内に入ったか

### STEP 3: 【医師】リアルタイム監視と最終同意の「ロック解除」
1. ライブ： 医師は患者の閲覧状況をリアルタイムで見れる
2. 見落とし判定: **患者が通り過ぎた（未達ではない）かつ注視不足の段落**を「読み飛ばし（黄色枠）」としてGeminiに内容を要約させて強調
    - 後半の未到達部分は無視し、ノイズを減らす
3. 説明: 医師が見落とし箇所を口頭説明し、患者がそこを注視すると、更新ボタンを押した後に黄色枠が消える
4. 医師は全ての見落とし個所がなくなったことを確認し、「最終同意を許可」ボタンをクリック

### STEP 4: 【患者】最終同意と証拠の固定（オンチェーン）
1. 患者の画面にリアルタイムで「最終同意」ボタンが有効化される
2. 患者が「最終同意」をクリックすると、最終ハッシュが Cloud Run へ送信される
3. 署名・記録: Cloud Run がハッシュを検証し、 Cloud KMS で署名。その後、 Firestore の永続データ層に保存
4. 完了: 患者画面に「同意照会番号」を表示し、**URLを即時無効化**。医師側でも**更新ボタンなどは押せなくなる**
5. 蓄積された永続データのハッシュを、定期的に**ブロックチェーンにアンカーとして記録**
